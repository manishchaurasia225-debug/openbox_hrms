# Master Execution Directive

## Objective

Build the entire Enterprise HR Operations Platform from start to finish using all documentation in the `.claude` directory.

Treat every document as part of a single specification.

Before starting any work, read every document.

---

## Execution Order

Always follow this order:

1. CLAUDE.md
2. project-rules.md
3. requirements.md
4. business-rules.md (if present)
5. architecture.md
6. backend-architecture.md
7. frontend-architecture.md
8. database.md
9. api-spec.md
10. coding-standards.md
11. security.md
12. performance.md
13. testing.md
14. deployment.md
15. implementation-roadmap.md
16. review-checklist.md
17. decisions.md
18. prompts.md

Never ignore any document.

---

## Execution Strategy

Work module by module.

Completely finish one module before beginning the next.

A module is complete only when:

- Implementation finished
- Tests pass
- Documentation updated
- Review checklist passed
- Git committed
- GitHub pushed

Then automatically continue to the next approved module.

---

## Autonomous Behaviour

Do not stop after every file.

Do not stop after every class.

Do not stop after every API.

Continue implementing until the entire module is complete.

---

## Continuous Improvement

While implementing:

- Improve architecture where appropriate.
- Improve performance.
- Improve security.
- Improve maintainability.
- Improve scalability.
- Improve code quality.

Never reduce functionality.

---


## Final Goal

Your responsibility is to build a production-ready enterprise HRMS with minimal user intervention.

Only stop if an Approval Required condition is encountered.
